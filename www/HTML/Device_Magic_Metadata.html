Data Sources and Notes

This table contains raw data uploaded from the device magic form which are then organized into their respective tables.

General Info

DMID: This is an autogenerated value that is created when a new record is added.
Submitted: The date and time the form was submitted from device magic (local time PST/PDT, YYYY-MM-DD).
Date Visit: The date the site visit occurred (YYYY-MM-DD).
Time Visit: The time the site visit occurred (local time PST/PDT).
SiteID: The site that was visited.
Technician and Technicians Other: List of technicians who were in the field.
Upstream Photo: Link to an upstream photo from the site.
Downstream Photo: Link to a downstream photo from the site.
Barrel Fill: Indicates if a reservoir refill was performed during the visit.
CF Event: Indicates if CF measurements were taken during the visit.
EC Sensor Change: Indicates if there were any EC sensor changes made to the network during the visit.

If barrel fill occurred

Volume Solution: This indicates how much solution was in the reservoir before it was refilled.
Salt Added : This indicates how much salt was added to the new solution mixture.
Salt Added : This indicates how much water was added to the new solution mixture.
Volume Depart : This indicates how much salt solution was in the reservoir upon departure.
Salt Remaining Site : This indicates how much salt remains on the site after the mixture was made.
Barrel Fill Notes: This contains any important notes about the refill.

If CF measurements were taken

Time Barrel Period: This indicates when the measurements occurred in relation to the a barrel fill event.
Trials CF: This indicates how many trials were done for each measurement.

If changes were made to the sensor network

Action: This tells us what action was performed.
(Add a sensor to the network, Remove a sensor from the network without replacement, Replace a sensor in the network with a new sensor.)
Reason: This indicates a reason for why the change occurred.

If a sensor was replaced

Sen R Removed Type (Other): This indicates the removed sensor's type.
Sen R Removed SN: This indicates the removed sensor's serial number.
Sen R New Type (Other): This indicates what the new sensor's type is.
Sen R New SN: This indicates the new sensor's serial number.
Sen R New RiverLoc (Other): This indicates the new sensor's location in the river.
Sen R New ProbeNum: This indicates the new sensor's probe numbe.

If a sensor was removed

Sen Remove Type (Other): This indicates the sensor's type.
Sen Remove SN: This indicates the sensor's serial number.
Sen Remove ProbeNum: This indicates the sensor's probe number.

If a sensor was added

Sen Add Type: This indicates the sensor's type.
Sen Add RiverLoc (Other): This indicates where in the river the sensor was placed.
Sen Add ProbeNum: This indicates the sensor's probe number.

General Notes

Notes Weather: The weather during the field visit.
Notes Repairs:Repairs and adjustments that were made during the site visit.
Notes ToDo: Things that need to be done at the site during future visits.
Notes Other: Notes about the visit that do not fit into the other categories.

What has been checked by the code

Visit Added: Indicates if the visit information has been added to the Field Visit table.
Barrel Added: Indicates if the barrel fill information has been added to the Barrel Fill table.
Sensor Added: Indicates if the sensor network change information has been added to the Sensor table.
CF Added: Indicates if the CF event information from the device magic form corresponds to what has been entered into the Calibration Event and Results tables.

Units of Data

Volume Solution --> volume rounded down to nearest 25L
Salt Added --> kg
Water Added --> volume rounded down to nearest 25L
Volume Depart --> volume rounded down to nearest 25L
Salt Remaining Site --> kg
