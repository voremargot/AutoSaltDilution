Data Sources and Notes

DocID (PK): This is an autogenerated value.
EventID (FK): This comes from the primary key of the Autosalt Summary table and tells us which event the documents relate to.
SiteID (FK): This comes from the primary key of the Autosalt Summary table and tells us which site and event the documents relate to.
Link: A google drive link that brings you to the discharge calculations excel sheet used to help calculate the average discharge values. Note that there have been several different versions of the excel sheet over the history of the project and some events have more than one excel sheet (depending on the number of sensors at the site).
Checked: Has the document been checked by a user? Many of these documents are automatically generated through code and need to be validated by the user.
Edits Made: When the document was checked where any edits made? Prior to May 2021, all documents are marked N which may be untrue for some documents. The lack of records prevents us from having more percise information prior to 2021.

Data Specifications

Checked --> Allowed values: Y(Yes), N(No)
Edits Made --> Allowed values: Y(Yes), N(No)

Data Constraints

1. If a document has been checked (i.e. Checked =Y) then Edits Made can not be null
2. When entering the Link into the database it must be entered in HTML format in order to display the link properly
